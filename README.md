# 🕐 WiFi Clock — ESP8266 + ST7920 128×64

Многофункциональные WiFi-часы на базе **ESP8266 (NodeMCU)** с графическим LCD дисплеем **ST7920 128×64**. Проект поддерживает отображение погоды, прогноза, постов Telegram-канала, данных с датчика температуры и настройку через встроенный веб-интерфейс.

\---

## ✨ Возможности

|Экран|Описание|
|-|-|
|🕐 Часы|Крупное время, дата, день недели, IP адрес|
|🌤 Погода|Температура, ощущаемая, влажность, ветер, иконка|
|📅 Прогноз|Прогноз на 3 дня с иконками|
|📰 Telegram|Последний пост публичного канала, бегущая строка|
|🌡 Датчик|Температура и влажность с DHT22/DHT11/SHT30|

* ⚙️ Полная настройка через **веб-интерфейс** (WiFi, погода, экраны, Telegram, датчик)
* 💾 Настройки сохраняются в **EEPROM**
* 📡 **OTA обновление** прошивки по WiFi — без подключения кабеля
* 🌙 Ночной режим с автоматическим затемнением по расписанию
* 📶 Режим точки доступа (AP) если WiFi недоступен — `192.168.4.1`

## 🔌 Схема подключения

### ST7920 → NodeMCU (Hardware SPI)

|Пин дисплея|Пин NodeMCU|GPIO|
|-|-|-|
|VCC|VIN|5V|
|GND|GND|—|
|CS / EN|D8|GPIO15|
|SID / MOSI|D7|GPIO13|
|SCLK / CLK|D5|GPIO14|
|**PSB**|**GND** ⚠️|SPI режим|
|A (подсветка+)|3.3V через резистор|—|
|K (подсветка−)|GND|—|

> ⚠️ \*\*Пин PSB обязательно подключить к GND\*\* — это переключает дисплей в SPI режим.  
> ⚠️ \*\*Питание VCC — 5V\*\* (через VIN NodeMCU), не 3.3V.

### Датчик DHT22 → NodeMCU (опционально)

|Пин датчика|Пин NodeMCU|
|-|-|
|VCC|3.3V|
|GND|GND|
|DATA|D4 (GPIO2) — по умолчанию|

## 🚀 Установка

### 1\. Добавить поддержку ESP8266 в Arduino IDE

`File → Preferences → Additional Board Manager URLs`:

```
https://arduino.esp8266.com/stable/package\_esp8266com\_index.json
```

Затем: `Tools → Board → Board Manager` → найти **esp8266** → Install.

### 2\. Настроить проект

Открой `wifi\_clock\_st7920.ino` и найди структуру `Config` в начале файла.  
Измени значения по умолчанию:

```cpp
char ssid\[32]   = "ВАШ\_WIFI";       // имя WiFi сети
char pass\[64]   = "ВАШ\_ПАРОЛЬ";     // пароль WiFi
char city\[32]   = "Moscow";          // город для погоды
char apiKey\[48] = "ВАШ\_OWM\_KEY";    // ключ OpenWeatherMap
char tg\_channel\[33] = "rf4map";      // Telegram канал (без @)
```

### 3\. Получить API ключ погоды

1. Зарегистрируйся на [openweathermap.org](https://openweathermap.org)
2. `My API Keys` → скопируй ключ
3. Вставь в поле `apiKey` (бесплатный тариф — до 1000 запросов/день)

### 4\. Загрузить прошивку

* **Board:** `NodeMCU 1.0 (ESP-12E Module)`
* **Upload Speed:** `115200`
* **Flash Size:** `4MB (FS:2MB OTA:\~1019KB)`

## ⚙️ Веб-интерфейс

После первой прошивки открой браузер и перейди по адресу устройства (IP отображается на экране часов).

|Вкладка|Настройки|
|-|-|
|🌐 WiFi|SSID, пароль, часовой пояс|
|🌤 Погода|API ключ, город, интервал, °C/°F|
|📺 Экраны|Включить/выключить каждый экран, время показа|
|📰 Telegram|Канал, скорость прокрутки, превью поста|
|🌡 Датчик|Тип (DHT22/DHT11/SHT30), GPIO пин, коррекция|
|📊 Статус|IP, RSSI, RAM, Uptime, текущая погода|

## 📡 OTA обновление прошивки

После первой загрузки через USB — все следующие обновления можно делать **по WiFi**:

1. Устройство должно быть включено и подключено к сети
2. Arduino IDE → **Tools → Port** → выбрать `wifi-clock.local`
3. Нажать **Upload**
4. Ввести пароль: `ota1234`

> 💡 Смени пароль OTA в строке `ArduinoOTA.setPassword("ota1234")` перед прошивкой.
